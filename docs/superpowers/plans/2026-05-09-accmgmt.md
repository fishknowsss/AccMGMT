# AccMGMT Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a small internal Runway account occupancy board for 40 unlimited-plan accounts.

**Architecture:** Vite React serves a single dashboard on Cloudflare Pages. Cloudflare Pages Functions expose a small JSON API, Cloudflare D1 stores accounts and bookings, and pure TypeScript domain helpers calculate current status and prevent overlapping reservations.

**Tech Stack:** Vite, React, TypeScript, Cloudflare Pages Functions, Cloudflare D1, Vitest, CSS.

---

### File Structure

- `package.json`: scripts and dependencies.
- `index.html`: Vite entry HTML.
- `src/main.tsx`: React entry.
- `src/App.tsx`: dashboard page shell.
- `src/components/account-dashboard.tsx`: client-side dashboard state, filters, dialogs, and table.
- `functions/api/accounts.ts`: list and create accounts.
- `functions/api/accounts/[id].ts`: edit account metadata.
- `functions/api/bookings.ts`: create immediate usage or future reservation.
- `functions/api/bookings/[id]/release.ts`: release active booking early.
- `lib/domain.ts`: account status, booking overlap, and validation helpers.
- `functions/_lib/repository.ts`: D1 persistence adapter.
- Account writes are handled by the Pages API without an operation PIN.
- `migrations/0001_init.sql`: D1 schema, indexes, and trigger-based no-overlap constraint.
- `wrangler.toml`: Cloudflare Pages build output and D1 binding.
- `tests/domain.test.ts`: TDD coverage for status and overlap behavior.
- `.env.example`: required environment variables.
