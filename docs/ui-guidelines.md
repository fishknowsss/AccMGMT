# AccMGMT UI 规范

## 设计目标

AccMGMT 应该像一个专业的内部资源调度工具，而不是普通后台模板。

参考方向：

- Linear：高密度、清爽、状态明确、少装饰。
- Airtable Interfaces：内部工具感，筛选和权限清晰。
- Resource Guru / Skedda：资源预约、可用资源、避免重复占用。

## 第一屏结构

桌面端第一屏应优先展示：

1. 页面标题和当前用户。
2. 账号池概览。
3. 搜索和筛选。
4. 账号资源矩阵。

禁止第一屏出现：

- 营销型 hero
- 说明文档式段落
- 复杂图表
- 复杂日历
- 账号详情卡堆叠
- 实现说明和调试说明

## 主导航

允许的主导航：

- Runway 看板
- 成员小组
- 账号设置

导航切换时，外层页面宽度不得闪变。

实现要求：

- sidebar 固定宽度。
- main 使用 `minmax(0, 1fr)` 或等效布局。
- 页面保留稳定滚动条 gutter。
- 切换视图只替换内容区，不改变应用骨架。

## Runway 看板

看板是核心视图。

必须包含：

- 账号池统计
- 搜索账号邮箱
- 状态筛选
- 小组筛选
- 续费筛选
- 找可用账号
- 账号资源矩阵

资源矩阵建议列：

- 账号
- 状态与时间
- 当前使用
- 续费
- 操作

注意：

- 不要做独立账号详情页。
- 不要把未来预约信息扩展成复杂日历。
- 下一预约可以作为状态副行或紧凑信息出现，但不要让表格横向臃肿。

## 账号设置

设置页是实际设置页，不是账号列表复读。

非编辑模式下可以展示：

- 当前用户
- 账号计划
- 账号数量
- 默认预约时长
- 续费提醒规则
- 结束权限规则

禁止展示：

- Mock 数据
- 本地会话
- 隐藏按键序列
- 脚本名
- 命令名
- 只影响当前页面
- 任意实现或调试说明

编辑模式下才展示账号编辑能力。

## 开发者模式 / 编辑模式

开发者模式的用户可见名称是 `编辑模式`。

规则：

- 只能在 `账号设置` 视图中开启。
- 普通 UI 不显示开启方式。
- 开启后只显示必要状态，不解释内部机制。
- 只允许编辑账号信息。
- 不影响普通成员在看板中的使用、预约、结束权限。

编辑模式允许：

- 新增账号
- 编辑账号邮箱
- 编辑账号编号
- 编辑续费日期
- 启用或停用账号

编辑模式禁止：

- 密码字段
- 多平台字段
- 额度字段
- 账号详情页
- 复杂审计页

## 文案规范

面向内部中文用户，自然直接。

按钮：

- 使用
- 预约
- 结束
- 保存
- 取消
- 找可用
- 新增账号

状态：

- 空闲
- 使用中
- 即将续费
- 需确认

空状态：

- 调整筛选条件。
- 先新增账号。

不要使用：

- 营销口吻
- 翻译腔
- “轻松管理”“智能高效”等泛泛文案
- tooltip 式废话

## 视觉规则

- 背景：浅色中性底。
- 内容最大宽度：1440px。
- 桌面端优先，移动端可横向滚动表格。
- 表格行高约 60 到 68px。
- 使用 tabular numbers 显示数字和时间。
- 状态色低饱和。
- 避免过多圆角、阴影、边框同时出现。
- 不使用装饰性图标填空。
- 不使用渐变球、bokeh、营销插画。

## 禁止项清单

任何 UI 改动完成后都要检查：

- 页面是否渲染了实现说明？
- 页面是否渲染了隐藏入口说明？
- 页面是否出现 mock、本地、脚本、命令等技术词？
- 是否新增了密码字段？
- 是否新增了额度字段？
- 是否新增了多平台字段？
- 是否新增了账号详情页？
- 是否新增了复杂日历？
- 普通成员是否仍能使用、预约、结束自己的使用？
- 编辑模式是否只影响账号信息编辑？
